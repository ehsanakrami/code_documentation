32// The question is: WHERE is the ORDER of the VARIABLES established?
33// I mean, on one hand there is an ORDER REGARDLESS of the FE family;
34//on the other hand there is an ORDER IN TERMS of FE.
35// The ORDER is important for defining the BLOCKS in the VECTORS and MATRICES.
36//In the initMeshToDof, i order QQ, LL and KK variables, BUT STILL YOU DO NOT KNOW
37// to WHICH QUANTITIES those VARIABLES are ASSOCIATED!
38// We should make a map that, for each SCALAR VARIABLE of EACH FE TYPE,
39// tells you TO WHICH QUANTITY it is ASSOCIATED.
40// The point is then you have SCALAR variables associated to SCALAR QUANTITIES,
41// or SCALAR VARIABLES associated to VECTOR QUANTITIES.
42// So you should associate SCALAR VARIABLES to COMPONENTS of QUANTITIES.
43// The first time when you start associating scalar variables to quantities
44// is when you set the BC's
45
46
47// ================================================================================
48// Now, the idea in the construction of the ELEMENT MATRIX and ELEMENT RHS is this.
49// You fill the element matrix, and each row (and column) of it will correspond
50// to a certain row of the global matrix.
51// HOW DO I KNOW THAT THIS CORRESPONDENCE is OK?
52// Well, this is given by the el_dof_indices.
53// every component of el_dof_indices tells me what is the GLOBAL ROW (and GLOBAL COLUMN)
54// So, if the length of your el_dof_indices is 10, then you'll have 10x10 elements to add to the global matrix,
55// given by all the possible couples "el_dof_indices[i] X el_dof_indices[j]"
56// TODO: Now, how do you know that the phi of one line is the correct phi, so it is the phi of THAT DOF?
57// I mean, how is the dependence on the REAL domain (NOT canonical) involved?
58// For the phi, the values are the same both on the canonical and on the stretched domain.
59// For the derivatives, I think the point is: you must pick the REAL dphidx in the SAME ORDER as you pick the CORRESPONDING DOFS.
60// Now, my point is: on a given row, are you sure that the code picks the correct dphidx?
